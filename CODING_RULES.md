# Coding Rules for Poker Engine

## Главная цель проекта

Мы создаём покерный движок Texas Hold'em на чистом Python.

Сначала делаем только ядро:
- карты;
- колода;
- игроки;
- стол;
- раздача;
- блайнды;
- банк;
- улицы игры.

Django, WebSocket, фронтенд и база данных будут подключаться позже.

## Главный принцип архитектуры

Движок должен быть независимым от интерфейса.

Нельзя смешивать:
- покерную логику;
- WebSocket;
- HTML;
- Django;
- базу данных;
- ввод с клавиатуры;
- вывод в консоль.

Покерный движок должен работать сам по себе и быть пригоден для тестов.

## Основные модули

### cards.py

Отвечает только за карты:
- Card
- Suit
- Rank

### deck.py

Отвечает только за колоду:
- создание 52 уникальных карт;
- перемешивание;
- выдача карт.

### player.py

Отвечает за состояние игрока:
- id;
- name;
- stack;
- hand;
- current_bet;
- folded;
- all_in.

### table.py

Отвечает за состояние стола:
- table_id;
- players;
- button_position;
- small_blind;
- big_blind;
- pot;
- community_cards;
- current_bet;
- street.

### engine.py

Отвечает за ход одной раздачи:
- start_hand;
- post_blinds;
- deal_hole_cards;
- deal_flop;
- deal_turn;
- deal_river.

## Правила кода

- Использовать Python 3.12+.
- Использовать type hints.
- Код должен быть простым и читаемым.
- Один класс — одна зона ответственности.
- Не делать больших методов.
- Не добавлять Django/WebSocket в ядро движка.
- Не добавлять оценку комбинаций, пока не готова базовая механика раздачи и ставок.
- Не добавлять сложные функции раньше времени.
- Каждый новый функционал должен сопровождаться тестами.

## Правила тестов

Тесты должны проверять:
- колода содержит 52 уникальные карты;
- игрок получает 2 карты;
- флоп содержит 3 карты;
- терн содержит 1 карту;
- ривер содержит 1 карту;
- блайнды корректно списываются со стеков;
- банк корректно увеличивается;
- current_bet равен большому блайнду после постинга блайндов.

## Правила логирования

В будущем все действия движка должны логироваться.

Лог должен показывать:
- кто сделал действие;
- что произошло;
- какая улица игры;
- какой банк;
- какая текущая ставка;
- какие изменения произошли у игроков.

Логирование нужно для будущей отладки Django/WebSocket.

## Важное правило

Codex не должен сразу переписывать весь проект.

Любая задача должна выполняться маленькими шагами:
1. изменить минимально нужные файлы;
2. добавить или обновить тесты;
3. объяснить, что изменено;
4. создать pull request.
